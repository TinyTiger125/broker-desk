# Skill: Output Preview UI

## When to use
- Output preview page changes
- Print/PDF-oriented layout decisions
- Document rendering hierarchy updates

## Objective
Keep generated output previews professional and document-like.

## Required Principles
1. Preview is a document canvas, not a dashboard card.
2. Use print-oriented structure (A4 mindset).
3. Preserve strong text hierarchy:
   - document title
   - section headers
   - field/value blocks
   - control metadata/footer
4. Keep clear preview controls:
   - zoom
   - print
   - download
   - source trace

## Checklist
- Is the page printable without visual breakage?
- Are legal/business sections clearly grouped?
- Is document control info visible (id/version/date/actor)?

# Skill: Workspace Admin UI

## When to use
- Dashboard/admin workspace layout changes
- Navigation, shell, card/table density decisions
- Large page composition refactors

## Objective
Keep pages looking like a serious business system for Japanese small/micro real-estate operators.

## Rules
1. Desktop-first productivity > decorative style.
2. Use stable layout primitives:
   - top summary band
   - operational list/table
   - right-side action/context panel when needed
3. Avoid over-carded dashboards with weak actionability.
4. Prefer readable hierarchy and short action paths.

## Checklist
- Is this page action-oriented?
- Are critical operations visible above the fold?
- Are KPIs tied to next actions?
- Does layout remain stable under JA/ZH/KO content length?

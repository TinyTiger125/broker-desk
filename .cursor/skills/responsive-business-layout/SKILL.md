# Skill: Responsive Business Layout

## When to use
- Responsive layout tuning
- Breakpoint changes for dashboard/table/detail pages
- Cross-device usability checks

## Objective
Enforce desktop-first productivity while keeping tablet/mobile usable.

## Policy
1. Desktop is primary workspace.
2. Tablet/mobile support high-frequency assistive tasks only.
3. Do not degrade desktop data density to mimic mobile-first apps.

## Mobile/Tablet Required Usability
- View details and linked records
- Add/update short records
- Check reminders/tasks
- Open output previews

## Checklist
- Are key desktop tables readable and actionable?
- Is there a responsive fallback for fixed shells/sidebars?
- Are action buttons and labels stable across locales?
- Are critical forms still usable on small screens?

# Broker Desk 开发总结（对齐产品经理）

## 版本范围
本轮完成了 MVP 核心流程 + 阶段 2 体验增强 + 数据持久化切换能力。

## 本轮目标
1. 让核心业务链路可跑通：客户 -> 跟进 -> 报价 -> 看板推进。
2. 强化操作效率：快速建档、拖拽推进、报价复用与打印。
3. 为上线准备数据基础：支持从内存模式切换到 PostgreSQL/Supabase。

## 已完成功能
### 1) 核心页面与路由
- Dashboard
- 客户列表（筛选/排序/快速操作）
- 新建客户 / 编辑客户
- 客户详情（左主内容 + 右侧摘要栏）
- 报价列表 / 报价新建 / 报价详情 / 报价打印页
- 跟进看板（Kanban）

### 2) 关键交互
- 顶部全局搜索、`+新建客户`、`+新建报价`
- 客户列表支持：阶段/用途/热度筛选 + 排序
- 客户详情支持：快速追加跟进记录、阶段更新
- 报价页支持：左输入右实时试算、摘要复制
- 报价详情支持：状态更新、版本复制（`v2/v3/...`）、打印入口
- 看板支持拖拽改阶段

### 2.1) 本轮新增产品钩子（重点）
- 报价页新增“客户可读摘要”双版本：简洁版 / 正式版，一键复制
- 报价页新增“试算提示”：首付偏低、月支出偏高、费用缺失、首期过高等异常提醒
- 报价详情页新增“对比模式”：同客户最近 5 条方案并排比较关键字段
- Dashboard 新增 `Today List（自动优先级）`：按规则自动筛出“今天必须跟进”的客户并展示原因
- 客户详情页新增阶段建议卡：基于带看记录/报价轮次给出一键阶段推进建议
- 报价创建与报价修订会自动写入客户时间线（follow-up note）

### 3) 流程闭环（已打通）
- 新建客户 -> 客户详情 -> 新建报价 -> 报价详情
- Dashboard/列表 -> 客户详情 -> 快速跟进
- 看板拖拽改阶段 -> 自动写入一条跟进记录（形成可追溯轨迹）

## 数据层升级（本轮重点）
### 双驱动数据仓库
- `memory`（原型快速验证）
- `postgres`（Supabase/PostgreSQL 持久化）

### 切换方式
通过 `.env`：
- `DATA_DRIVER=memory`
- `DATA_DRIVER=postgres` + `DATABASE_URL=...`

### 设计结果
- 页面与 Server Actions 不需要改调用方式
- 数据源通过适配器切换，便于后续环境迁移
- 新增统一健康检查 API：`/api/health/data`（可快速确认当前驱动与连接状态）

### 已完成验证
- `memory` 模式实测返回：`{"ok":true,"driver":"memory"}`
- `postgres` 模式待接入真实 `DATABASE_URL` 后可同接口验证
- 页面烟测：Dashboard 优先级区块、报价对比模式、报价预览页均可正常渲染

## 当前边界（产品确认项）
1. 当前为单用户工作台语义（多团队权限未开启）
2. 自动消息触达、合同管理、会计对接暂不在 MVP
3. 报价字段已覆盖主流程，但仍建议与真实经纪人业务再做一轮字段校准

## 下一阶段建议（按优先级）
1. 接入真实 Supabase 项目并导入 20-50 条真实样本数据验证
2. 看板拖拽加入权限与操作日志筛选（谁在什么时间改了阶段）
3. 报价打印模板升级为正式客户可发送样式（含品牌抬头/页脚）
4. 增加基础统计：阶段转化漏斗、平均推进时长、逾期跟进率

### 3) Dashboard 法定提醒
- 新增“法定対応アラート”区块
- 覆盖：媒介契约到期提醒、35条未记录、37条未记录、AML待确认、个资同意缺失

### 4) 自动化填写准备（降低录入复杂度）
- 新增新建客户页模板预填：
  - `居住用ヒアリング`
  - `投資用ヒアリング`
  - `法定対応開始`
- 通过 URL 参数模板化注入默认值，为后续 OCR/聊天记录自动抽取填充预留入口
- 新增“ヒアリングメモ自動抽出（ルールベース）”：
  - 从自由文本抽取：用途、预算区间、预算种别、区域优先级、贷款预审、阶段、温度感、媒介契约、AML、个资同意线索

- 新增自动化接口：
  - `POST /api/clients/intake/parse`
  - 入参：`{ text: string }`
  - 出参：`recommendedTemplate` + `defaults` + `extracted`

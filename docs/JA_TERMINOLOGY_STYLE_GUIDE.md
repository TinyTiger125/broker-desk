# 日本市場向け用語統一ガイド（Broker Desk）

更新日: 2026-03-27
対象: 画面文言 / APIエラーメッセージ / 監査ログ文言

## 1. 目的
本ガイドは、日本の不動産仲介実務で違和感が出やすい用語を統一し、
画面・帳票・ログで同一語彙を使うための運用基準です。

## 2. ステージ表記（必須）
- `lead`: 新規受付
- `contacted`: 初回接触済み
- `quoted`: 提案送付済み
- `viewing`: 内見済み
- `negotiating`: 申込・条件調整
- `won`: 成約
- `lost`: 見送り

## 3. 見積ステータス表記（必須）
- `draft`: 作成中
- `sent`: 送付済み
- `revised`: 再提案

## 4. サマリー文言（必須）
- 短文 → **簡潔版**
- 正式文 → **詳細版**
- コピー文言: 「提案文をコピー」 / 「コピー済み」

## 5. 金額項目の表記ルール（必須）
- 管理費・修繕積立金は **月額入力** とする。
- 画面表示は「管理費（月額）」「修繕積立金（月額）」を使用。
- 月次総支出は「月々返済 + 管理費（月額） + 修繕積立金（月額）」で計算。

## 6. 印刷提案書の表記ルール（必須）
- ヘッダ表記: 「ブローカーデスク 提案書」
- 税関連項目: 「税金・諸費用」
- 注意書き: 「税金・登記費用・保険料」

## 7. 使用禁止（置換対象）
- 新規リード → 新規受付
- 見積提出済み → 提案送付済み
- 内見実施済み → 内見済み
- 失注 → 見送り
- 短文 / 正式文 → 簡潔版 / 詳細版
- 管理費（年）/ 修繕積立金（年） → 管理費（月額）/ 修繕積立金（月額）
- 税費用 → 税金・諸費用

## 8. 運用
- 新規画面追加時は本ガイドの語彙のみ使用。
- 文言修正PRでは、必ず「使用禁止語」の混入有無をチェックする。
- 監査ログ・通知文言も同一基準を適用する。
- CI/ローカル確認コマンド: `npm run test:ja-terms`

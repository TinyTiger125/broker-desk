# 开发总结（2026-03-27）

## 1. 本轮目标
将 Broker Desk MVP 从“页面骨架”推进到“可本地演示、可验证核心流程、具备专业工作台特征”的状态。

## 2. 已完成范围

### 2.1 任务与合规流程落地
- 新增任务状态流转：`pending / done / canceled`
- 支持任务延期（重设 due date）
- Dashboard 合规提醒支持两类动作：
  - 直接任务化
  - 直接标记“已解消”

### 2.2 通知中心与审计日志
- Dashboard 新增“通知センター”
- 新增“監査ログ（最新8件）”展示
- 关键动作写入审计日志（客户创建/编辑、跟进、阶段变更、报价创建/复制/状态变更等）

### 2.3 权限与数据安全（MVP 级）
- 服务端 action 增加客户归属校验（ownership check）
- Board 阶段变更 API 增加 owner guard
- 阻断越权修改路径

### 2.4 报价专业化
- 报价打印页升级为更专业提案样式（版本、日期、费用结构、月度试算、留意事项）
- 报价页保留实时试算能力

### 2.5 自动化录入（Autofill 2.0）
- 新建客户页支持置信度阈值
- 低于阈值字段不自动写入
- 增加“反映预览”
- 增加“模板字段冲突候选”提示

### 2.6 数据层与文档
- Memory / Postgres 双实现同步补齐
- Postgres schema 增加 `audit_logs` 与任务/审计索引
- 健康检查接口补充 `checkedAt`、`app`
- 补充回归脚本与运行文档

## 3. 日本市场本地化状态
- 前台核心业务文案已日文化（Dashboard、跟进、报价、提醒、看板等主路径）
- 业务表达已偏向日本不动产仲介语境（如追客、提案、法定対応提醒）

## 5.1 日文术语统一（日本市场表达校正）
- 统一阶段语义：`新規受付 / 初回接触済み / 提案送付済み / 内見済み / 申込・条件調整 / 成約 / 見送り`
- 统一报价状态：`作成中 / 送付済み / 再提案`
- 统一摘要表达：`簡潔版 / 詳細版`
- 统一费用口径：管理费与修繕積立金改为“月額入力”，月次支出按月额直接计入
- 新增术语规范文档：`docs/JA_TERMINOLOGY_STYLE_GUIDE.md`
- 新增术语自动检查：`npm run test:ja-terms`（`scripts/check-ja-terms.sh`）

## 5.2 CI 自动化验收
- 新增 GitHub Actions：`.github/workflows/ci.yml`
- push / pull request 自动执行：
  1. 术语检查（`test:ja-terms`）
  2. 代码风格检查（`lint`）
  3. 构建检查（`build`）
  4. 启动服务 + 回归脚本（`test:regression`）

## 5.3 产出物标准化升级（日本市场）
- 提案详情页新增“標準出力テンプレート”入口
- 输出页支持 4 类模板：
  - 購入提案書
  - 費用見積明細書
  - 資金計画書（ローン試算）
  - 試算前提条件説明書
- 每份文书补齐文档控制信息（文書番号 / 版数 / 発行日 / 作成主体）
- 增加统一承认栏、免责条款、对外说明格式
- 新增产出物设计与审计文档：
  - `docs/OUTPUT_PACKAGE_STANDARD_JP.md`
  - `docs/OUTPUT_AUDIT_JP_2026-03-27.md`

## 5.4 模板调整中心（可运维化）
- 新增页面：`/settings/output-templates`（帳票テンプレート）
- 支持统一调整：
  - 公司信息抬头
  - 四类文书标题
  - 文书区分
  - 留意事項三段文本
  - 三个显示开关（承认栏 / 法定対応サマリー / 借入残高推移表）
- 支持“一键恢复日本标准模板”
- 打印页已改为读取模板中心配置，保存后即时反映
- Postgres 新增 `output_template_settings` 持久化表

## 6. 上线前建议（下一阶段）
1. 术语统一：建立日文术语字典并做全量替换检查
2. 数据持久化切换：默认启用 Postgres（含迁移策略）
3. 认证增强：最小 RBAC（店铺/成员）
4. 通知触达：邮件或 Line webhook（任务临期提醒）
5. 打印模板：按日本经纪现场再打磨 1 版正式提案模板

## 7. 风险与边界说明
- 当前为 MVP 强化版，目标是“可用与可演示”，非生产完备版
- 审计与权限已做基础防护，但尚未覆盖复杂组织权限模型
- 自动填充规则可用，但仍建议结合真实门店数据继续调阈值

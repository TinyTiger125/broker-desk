# Foundation UX Hardening Tasklist

## 1. 文档状态
- 版本：v0.1
- 状态：本阶段完成（可进入增强层）
- 范围：基础功能体验优化（不包含 AI 实施）

## 2. 执行边界
- 本清单仅覆盖基础体验优化，不扩展新业务模块。
- 所有任务遵循 `Claude.md` 第14节收敛规则。
- 执行前需二次确认优先级与里程碑顺序。

## 3. 任务列表（按目标拆解）

### 已落地（2026-03-31）
- UX-03（部分）：核心表单已接入自动保存 + 草稿恢复 + 草稿放弃
- UX-08（部分）：服务请求状态操作支持 `returnTo` 保留上下文
- UX-10（部分）：关键页面统一成功反馈（flash banner）
- UX-01（部分）：导入中心补齐导入任务创建入口 + 附件完整登记字段
- UX-04（部分）：全局搜索接入跨实体检索（导航顶部）
- UX-05（部分）：服务请求支持批量状态更新
- UX-06（部分）：输出中心接入生成前质量校验并阻断缺项输出
- UX-07（部分）：输出缺项可修复清单（问题码 -> 可读文案）
- UX-09（部分）：关键列表页增加空状态引导与操作入口
- UX-02（补齐）：表单最近值复用（reuse）能力已接入核心表单
- UX-05（补齐）：批量操作覆盖到 Contracts + 选中导出（Properties/Parties/Contracts）
- UX-10（补齐首批）：服务请求状态支持 Undo

### UX-01 导入闭环提速（P0）
- 目标：导入任务从创建到可用数据的链路显著缩短。
- 页面范围：`/import-center`
- 后端范围：`createImportJobAction` / `autoMapImportJobAction` / `updateImportJobMappingAction` / `resolveImportValidationAction`
- 任务项：
  - `UX-01-T01` 将导入流程统一成可见 4 步状态（创建、映射、校验、完成）
  - `UX-01-T02` 在导入页增加“当前阻塞点”提示（缺映射、缺必填、格式错误）
  - `UX-01-T03` 增加“一键进入下一步”按钮，减少页面往返
  - `UX-01-T04` 增加导入耗时统计字段（创建到完成）
- 验收：
  - 用户可在单页完成主要流程，不需要跨页跳转
  - 回归脚本可覆盖导入链路关键步骤

### UX-02 表单自动复用（P0）
- 目标：降低重复录入，优先复用历史字段。
- 页面范围：`/properties` `/parties` `/contracts` `/service-requests` `/output-center`
- 后端范围：新增“最近使用字段”读取接口/action（若采用 server action 可先内聚）
- 任务项：
  - `UX-02-T01` 定义复用字段白名单（按实体）
  - `UX-02-T02` 表单初始化优先加载：当前实体值 > 最近有效值 > 默认模板
  - `UX-02-T03` 提供“一键清空复用值”入口
  - `UX-02-T04` 记录复用命中率（用于后续评估）
- 验收：
  - 关键表单可自动带出常用字段
  - 用户可明确识别“自动填充”来源并覆盖

### UX-03 自动保存与草稿恢复（P0）
- 目标：避免录入中断导致数据丢失。
- 页面范围：`/import-center` `/contracts` `/output-center`（首批）
- 后端范围：草稿存储（可先 local + server 混合）
- 任务项：
  - `UX-03-T01` 定义草稿粒度（页面级 + 表单级）
  - `UX-03-T02` 30s 自动保存 + 离开页面前保存
  - `UX-03-T03` 回访页面时展示“恢复草稿”提示
  - `UX-03-T04` 提供“放弃草稿”动作
- 验收：
  - 页面异常刷新后能恢复最近编辑内容
  - 草稿恢复不覆盖已提交正式数据

### UX-04 全局搜索与跨实体跳转（P0）
- 目标：单入口检索所有核心实体并快速跳转。
- 页面范围：全局导航搜索（`app-nav`）
- 后端范围：新增统一搜索 API（实体分组返回）
- 任务项：
  - `UX-04-T01` 定义搜索索引字段（物件/主体/合同/服务请求/输出物）
  - `UX-04-T02` 下拉结果分组展示 + 高亮匹配
  - `UX-04-T03` 支持快捷跳转到详情/聚焦位置
  - `UX-04-T04` 空结果显示引导动作（新建入口）
- 验收：
  - 支持至少 5 类实体统一检索
  - 搜索到达目标页面的中位点击步数 <= 2

### UX-05 批量操作（P0）
- 目标：减少重复点击，提升运营效率。
- 页面范围：`/contracts` `/service-requests` `/parties` `/properties`（按优先级分批）
- 后端范围：批量 action/API（状态变更、归档、导出）
- 任务项：
  - `UX-05-T01` 表格支持多选与全选
  - `UX-05-T02` 批量状态更新 action
  - `UX-05-T03` 批量归档 action
  - `UX-05-T04` 批量导出 action（CSV/PDF入口）
- 验收：
  - 批量操作有预检（非法状态组合阻断）
  - 批量结果有成功/失败明细反馈

### UX-06 输出前质量闸门（P1）
- 目标：输出物生成前确保字段与文控完整。
- 页面范围：`/output-center` `/quotes/[id]/print`
- 后端范围：输出前校验函数（可复用）
- 任务项：
  - `UX-06-T01` 定义四类输出物必填字段规则
  - `UX-06-T02` 定义文控规则（文档编号、模板版本、发行日期、责任人）
  - `UX-06-T03` 输出前阻断并返回可修复项
  - `UX-06-T04` 保存校验快照到输出记录
- 验收：
  - 缺关键字段时禁止正式生成
  - 每次生成保留可审计校验结果

### UX-07 错误可修复化（P1）
- 目标：错误提示可行动，而不是只提示。
- 页面范围：全站表单页 + 导入页 + 输出页
- 后端范围：错误码规范化
- 任务项：
  - `UX-07-T01` 统一错误码和文案映射
  - `UX-07-T02` 为每类错误绑定修复入口（按钮/深链）
  - `UX-07-T03` 支持“定位到字段”与“定位到步骤”
  - `UX-07-T04` 记录错误修复完成率
- 验收：
  - 关键错误都能一键进入修复位置
  - 用户无需手动猜测修复路径

### UX-08 列表状态记忆（P1）
- 目标：返回列表后保留用户上下文。
- 页面范围：`/properties` `/parties` `/contracts` `/service-requests` `/output-center`
- 后端范围：无硬依赖（URL + local state 即可）
- 任务项：
  - `UX-08-T01` 将筛选/排序/分页参数 URL 化
  - `UX-08-T02` 返回列表时保留 scroll 与 focus 行
  - `UX-08-T03` 提供“重置视图”入口
- 验收：
  - 详情页返回后列表状态不丢失
  - 用户无需重复筛选定位

### UX-09 空状态与首用引导（P1）
- 目标：无数据场景下仍能引导完成第一步动作。
- 页面范围：8个主模块页面
- 后端范围：无
- 任务项：
  - `UX-09-T01` 每页定义空状态文案与推荐动作
  - `UX-09-T02` 提供首用快捷入口（导入样例/新建模板/快速创建）
  - `UX-09-T03` 建立“首个可见成果”路径提示
- 验收：
  - 空页面不出现“纯留白”
  - 新用户可在 3 步内完成首个有效操作

### UX-10 统一反馈与可撤销（P1）
- 目标：结果反馈一致、关键操作可回退。
- 页面范围：全站
- 后端范围：操作日志 + 软撤销支持
- 任务项：
  - `UX-10-T01` 统一 toast/inline 反馈规范
  - `UX-10-T02` 关键动作增加“撤销”窗口（短时）
  - `UX-10-T03` 操作日志可查询（至少管理员可见）
  - `UX-10-T04` 定义不可撤销动作清单
- 验收：
  - 用户可明确知道动作是否成功与下一步
  - 关键误操作可在窗口期内撤回

## 4. 里程碑映射（6周）

### Milestone A（第1-2周）
- 包含：UX-02 / UX-03 / UX-08
- 交付重点：减少重复录入 + 防中断 + 状态不丢失

### Milestone B（第3-4周）
- 包含：UX-04 / UX-05 / UX-01
- 交付重点：检索效率 + 批量效率 + 导入全链路提速

### Milestone C（第5-6周）
- 包含：UX-06 / UX-07 / UX-09 / UX-10
- 交付重点：输出质量、错误修复、首用引导、操作可靠性

## 5. 执行前检查单（每个任务开工前）
1. 是否满足 `Claude.md` 14.1 准入标准（至少 3/5）
2. 是否有明确验收口径（可测量）
3. 是否定义了不做范围（避免扩展）
4. 是否评估了 i18n 三语一致性影响
5. 是否具备回归脚本覆盖点

## 6. 指标追踪（每周复盘）
1. 首份合格输出物生成时长
2. 人工填写字段数量
3. 输出物退回/修改率
4. 导入闭环完成时长（辅助指标）
5. 批量操作使用率（辅助指标）

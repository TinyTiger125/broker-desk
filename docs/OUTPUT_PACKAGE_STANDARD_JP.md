# 日本市场产出物标准设计（v1.0）

日期: 2026-03-27  
适用对象: 日本不动产卖买仲介（单店/小团队）

## 1. 设计目标
产出物必须满足三件事：
1. 对客户“可直接发送”，形式正式
2. 对经纪“可直接解释”，结构清晰
3. 对公司“可审计追踪”，版本与责任明确

## 2. 标准产出物清单

1. 購入提案書（`proposal`）
- 目的: 给客户的综合说明
- 核心内容: 方案摘要、取得时费用、月次负担、担当评论、免责与确认栏

2. 費用見積明細書（`estimate_sheet`）
- 目的: 对费用结构进行可审阅化说明
- 核心内容: 费用分项、算定依据、合计口径、月额维护成本

3. 資金計画書（ローン試算）（`funding_plan`）
- 目的: 说明融资结构与还款压力
- 核心内容: 借入条件、月供、月次总支出、借入余额推移

4. 試算前提条件説明書（`assumption_memo`）
- 目的: 固化“本次试算前提”并防止争议
- 核心内容: 前提条件、不确定项、确认事项清单、签核栏

## 3. 文书统一结构（所有模板必须包含）

### 3.1 文书头（Document Control）
- 文書番号（自动生成）
- 版数（Ver.x）
- 発行日
- 区分（社外提出用 / 社内確認用）

### 3.2 当事方信息块
- 宛先（顾客名）
- 提案名
- 对象物件
- 公司信息（公司名、部门、担当、地址、电话、邮箱、免许番号）

### 3.3 统一脚注
- 概算性质声明
- 最终条件以法定文书与契约文书为准
- 版本确认提示

### 3.4 承认栏
- 担当者確認
- 顧客確認

## 4. 字段口径标准（强制）
- 管理费: 月额
- 修繕積立金: 月额
- 月次総支出: `月々返済額 + 管理費（月額） + 修繕積立金（月額）`
- 取得関連費用合計: `仲介手数料 + 税金・諸費用 + その他費用`
- 契約時必要自己資金（概算）: `頭金 + 取得関連費用合計`

## 5. 文件命名与编号规则
- 编号规则: `BD-{DOC_TYPE}-{YYYYMMDD}-{TOKEN}`
- 文件建议名:
  - `購入提案書_{顧客名}_{YYYYMMDD}.pdf`
  - `費用見積明細書_{顧客名}_{YYYYMMDD}.pdf`
  - `資金計画書_{顧客名}_{YYYYMMDD}.pdf`
  - `試算前提条件説明書_{顧客名}_{YYYYMMDD}.pdf`

## 6. 系统落地位置
- 模板枚举与编号: `src/lib/output-doc.ts`
- 多模板输出页面: `src/app/quotes/[id]/print/page.tsx`
- 模板入口（提案详情页）: `src/app/quotes/[id]/page.tsx`
